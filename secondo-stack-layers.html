Retell AI — AI Call Agent

The persistent AI participant in the phone call • Custom tool (vault-lookup) • THE patent claim
⚠ DEMO BLOCKER Code Scaffolded

What it does

When Secondo places an outbound call, Retell’s AI agent joins the call as a persistent participant. It listens to the service provider’s rep, and when it needs information from the user’s vault, it calls the vault-lookup Edge Function as a “custom tool.” This real-time, mid-call document retrieval is patent claim P1-C1 — the independent claim that the entire provisional filing rests on.

Retell connects to Twilio (BYO) to dial out to real PSTN numbers.

Real phone network access • BYO Twilio subaccount under MBACIO parent • Retell dials through this
⚠ DEMO BLOCKER

What it does

Provides the actual phone network (PSTN) connectivity. Retell uses “BYO Twilio” — you supply your own Twilio subaccount credentials and a purchased phone number, and Retell dials outbound calls through it. Without Twilio, Retell can’t reach any real phone number.

Picovoice Porcupine — Wake Word

On-device, always-listening “Hey Secondo” detection • Zero audio sent to cloud
Installed, Not Configured Not Blocking Demo

What it does

Runs a tiny neural network on-device to listen for a specific wake phrase. When “Hey Secondo” is detected, it fires an event that triggers the voice pipeline. All audio processing happens entirely on the device — no audio ever leaves the phone, making it HIPAA-friendly and latency-free.

For April 28: the manual “Listen” button in the Calls tab is a perfectly acceptable demo path. Counsel doesn’t need to see wake-word activation to understand the patent claim.

whisper.rn — On-Device Speech-to-Text

Whisper.cpp port for React Native • Fully offline STT • HIPAA-compliant (no cloud audio)
Installed, Not Configured iOS STT Fallback Available

What it does

Transcribes the user’s spoken voice to text entirely on-device using OpenAI’s Whisper model (via whisper.cpp). Supports multiple model sizes — whisper-tiny (~75 MB) is fast enough for real-time use on modern iPhones. Used in VoiceSession.ts after wake word detection (or after “Listen” button tap).

Demo fallback: iOS native SpeechRecognizer (built into VoiceSession as a fallback path) activates automatically if the whisper model is not loaded. Slightly less accurate, but demo-ready.

MIT — mybigday/whisper.rn

Cost

whisper.rn — 100% free, forever
Free
MIT license. On-device — zero API calls, zero usage fees, zero per-minute charge. OpenAI released Whisper model weights under MIT in 2022: also free commercially. Model files: whisper-tiny GGML ~75 MB, whisper-base ~142 MB.
iOS Native SpeechRecognizer (demo fallback)
Free
Built-in iOS API. Uses Apple’s servers (not HIPAA-long-term). Active in VoiceSession.ts as automatic fallback if whisper model file isn’t loaded. Demo-ready today.

whisper.rn is a key competitive advantage vs. cloud-STT alternatives (Deepgram $0.0043/min, AssemblyAI $0.0065/min, Google STT $0.006/min). On-device = $0/min at any volume.

Claude API (Anthropic) — Intent & Post-Call LLM

Intent parsing Edge Function • Action item extraction from call transcript • Not blocking demo
Code Written Regex Covers Demo

What it does

Used in two places in the architecture:

1. Intent Edge Function — takes voice transcription text, calls Claude to parse structured intent and generate tool calls (e.g. “Call Geico about my claim” → {intent: "call", provider: "Geico", topic: "auto claim"}). Stronger than regex for ambiguous phrasing.

2. CallAgent.finalizeCall() — after a call ends, Claude extracts structured action items from the transcript (“Send policy docs by Friday” → calendar event + ClickUp task).

For demo: the regex intent parser in calls.tsx covers “Call Geico” without needing the Claude Edge Function.

Proprietary — anthropic.com

Cost

Claude Sonnet 4.5 (target model)
$3 / M input
$15 / M output
A typical intent parse: ~200 tokens in + ~100 tokens out = $0.0009 per call. Action item extraction: ~2,000 tokens = $0.009 per call. Demo-scale: cents, not dollars.
Demo fallback (regex)
Free
Regex intent parser in calls.tsx handles “Call [name]” patterns without any API calls. Suitable for April 28 demo flow.

  • CallAgent.finalizeCall() extraction logic written
  • Regex intent parser wired in calls.tsx (demo path)
