Three alternatives. Ranked by demo quality, implementation time, and risk.
Option How it works Demo quality Build time Risk Verdict
A — Manual "Listen" button
(already exists)		 User taps the button, speaks, STT runs. No wake word at all. OK for demo 0 hours None USE NOW as default
B — @react-native-community/voice
(Apple Speech Framework)		 Continuous listening via iOS Speech. Detect "Secondo" keyword in transcript. No API key, no Porcupine. Uses Apple's own STT engine. Near-identical to Porcupine 4–6 hours Low — mature package RECOMMENDED Plan B
C — expo-speech + polling
(continuous whisper.rn)		 Run whisper.rn in a short-burst loop (3s clips). Detect "Secondo" or "hey secondo" in transcript. Same STT already in the app. Good if latency acceptable 6–8 hours Battery drain, latency Fallback to B
Recommendation: Ship Plan B (Option B) this week regardless of Picovoice.

@react-native-community/voice uses iOS Speech Recognition (the same engine Siri uses). It is free, no API key, works offline for keyword detection, and ships in production apps. Install today, detect "Secondo" in the continuous transcript stream, trigger VoiceSession. The patent claim language says "voice activation" — it does not require Picovoice specifically.
