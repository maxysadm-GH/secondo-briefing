Patent Application #1

Persistent AI Participant in Voice Communications with Context-Aware Document Retrieval

Abstract

A computer-implemented system and method for providing an artificial intelligence agent that participates as a persistent entity in voice telephone communications, wherein the AI agent maintains real-time access to a user's encrypted document vault during the call, can retrieve and vocalize information from stored documents in response to natural language queries, initiates outbound calls on the user's behalf including automated Interactive Voice Response (IVR) navigation, performs warm transfer to conference the user with a live agent, provides live transcription with automated action item extraction, and generates post-call summaries with calendar event creation. The system maintains a persistent knowledge base (backend persona) about the user's stored documents, preferences, and historical interactions to provide contextually relevant assistance throughout the call lifecycle.

Technical Field

The present invention relates generally to voice communication systems enhanced with artificial intelligence, and more particularly to systems and methods for providing a persistent AI participant in telephone calls that can retrieve and present information from an encrypted document storage system in real-time during active voice communications.

Background of the Invention

Modern consumers frequently need to contact service providers (insurance companies, utilities, healthcare providers, government agencies) by telephone to resolve questions about their accounts, policies, and services. These calls typically require the consumer to have specific documents and information readily available — policy numbers, account details, dates, coverage amounts, and other data stored across various physical and digital locations.

Current voice assistants (Siri, Alexa, Google Assistant) can initiate phone calls but do not participate in the call, cannot access stored documents during the call, and provide no contextual assistance once the call is connected. Note-taking and transcription applications can record calls but do not actively participate or retrieve relevant documents in real-time.

There exists a need for a system that bridges the gap between document storage and voice communications — an AI agent that knows what documents the user has stored, can join telephone calls as a persistent participant, and can retrieve and vocalize specific information from those documents in real-time when questions arise during the call.

Detailed Description of the Preferred Embodiment

The invention comprises several interconnected subsystems operating on a mobile computing device with optional cloud processing capabilities:

1. Call Initiation Subsystem

Upon receiving a voice command (e.g., "Secondo, call Geico about my claim"), the system: (a) identifies the target entity from the user's contact database or stored documents; (b) retrieves relevant documents associated with that entity (insurance policy, claim numbers, account details); (c) initiates an outbound telephone call via programmable voice API (Twilio/Retell AI); (d) navigates IVR menu systems using DTMF tone generation and speech recognition to select appropriate department/options; (e) monitors hold queue status and notifies the user when a live agent is available; and (f) performs warm transfer by conferencing the user into the active call while the AI remains as a persistent participant.

2. Persistent AI Participant Subsystem

Once the call is active (whether user-initiated or AI-initiated), the AI agent operates as a persistent third participant on the call. The AI agent: (a) maintains a real-time audio stream of the conversation; (b) performs continuous speech-to-text transcription; (c) monitors for triggers that indicate the user needs document-based information (e.g., "What's my policy number?", "When did I file the claim?"); (d) retrieves relevant data from the encrypted document vault; (e) presents information to the user via text overlay on the mobile device screen or via synthesized speech injected into the call audio; and (f) maintains awareness of the conversation context to proactively surface relevant documents before the user explicitly requests them.

3. Backend Persona Knowledge System

The AI maintains a persistent knowledge base ("backend persona") comprising: (a) an index of all documents stored in the user's encrypted vault, with extracted metadata (dates, amounts, names, policy numbers, account numbers); (b) entity relationship mappings (e.g., "Geico" is linked to auto insurance policy, claim #12345, and three prior call transcripts); (c) user preferences and historical patterns (preferred communication style, frequently referenced documents, common queries); and (d) action item history from previous calls with the same entity.

4. Post-Call Processing Subsystem

Upon call termination, the system automatically: (a) generates a structured summary of the call including participants, duration, topics discussed, and outcomes; (b) extracts action items with associated dates and deadlines; (c) creates calendar events for scheduled follow-ups, appointments, or deadlines mentioned during the call; (d) updates the entity relationship model with new information obtained during the call; (e) stores the call transcript in the appropriate room of the document vault; and (f) creates reminder notifications for pending action items.

Patent Application #2

Voice-Activated Multi-Room Encrypted Document Vault with Natural Language Query

Abstract

A computer-implemented system and method for organizing, storing, and retrieving personal documents using a voice-activated encrypted document vault organized as a hierarchical "Mind Palace" comprising multiple categorized rooms. The system provides voice-activated optical character recognition (OCR) capture of physical documents with an AI-guided categorization wizard, natural language querying that searches across rooms and synthesizes voice responses from structured document data, local-first zero-knowledge encrypted storage ensuring document privacy, and cross-room intelligence that identifies and retrieves related documents from multiple rooms in response to a single query. The Mind Palace metaphor provides an intuitive organizational framework that maps to how users naturally think about different areas of their lives.

Technical Field

The present invention relates generally to document management systems with voice interfaces, and more particularly to systems and methods for voice-activated capture, encrypted hierarchical organization, and natural language retrieval of personal documents using a room-based metaphor with cross-category intelligence.

Background of the Invention

Modern individuals manage an increasing volume of critical personal documents across multiple life domains: personal identification, financial records, insurance policies, medical records, employment documents, family records, pet care documents, property records, and more. These documents are typically scattered across physical filing cabinets, email attachments, cloud storage folders, camera rolls, and various application silos.

Existing document management solutions (Notion, Evernote, Google Drive, Apple Notes) are primarily text-based interfaces that require manual typing, folder navigation, and visual search. They do not provide voice-first interaction, do not offer an intuitive hierarchical organization metaphor, and do not support voice-activated document capture with guided categorization.

There exists a need for a document management system that is voice-first, uses an intuitive organizational metaphor (rooms in a Mind Palace), provides zero-knowledge encryption for privacy, and enables natural language retrieval of specific data points from stored documents without requiring the user to remember which folder or category contains the target document.

Detailed Description of the Preferred Embodiment

1. Mind Palace Architecture

The document vault is organized as a hierarchical structure of "rooms," each representing a life domain. Default rooms include Personal, Work, Family, and Pets, with the user able to create custom rooms. Each room contains sub-categories (e.g., Personal → Insurance → Auto, Personal → Housing → Lease). Documents within rooms are stored with extracted metadata including dates, monetary amounts, entity names, expiration dates, and custom tags. The room metaphor provides spatial memory cues that improve recall and retrieval accuracy compared to flat folder hierarchies.

2. Voice-Activated OCR Capture System

Document capture is initiated by voice command or camera activation. The system: (a) captures an image of the physical document via the device camera; (b) performs on-device OCR using a machine learning model (e.g., Google ML Kit) to extract text content; (c) initiates a voice-guided categorization wizard that asks the user a series of natural language questions ("Which room does this belong to?", "Is this an insurance document?", "When does it expire?"); (d) auto-suggests categorization based on OCR content analysis (e.g., detecting "GEICO" and "Policy Number" suggests Personal → Insurance → Auto); (e) extracts and indexes key data points (dates, amounts, names, numbers); and (f) encrypts and stores the document with full metadata in the appropriate room.

3. Natural Language Query Engine

The query system processes natural language questions and retrieves specific data points: (a) the user issues a voice query (e.g., "When does my lease expire?"); (b) the system parses the query using natural language understanding to identify the intent (retrieve expiration date) and the target entity (lease); (c) the system searches the document index across all rooms to locate the relevant document; (d) the system extracts the specific data point responsive to the query; (e) the system synthesizes a natural language voice response (e.g., "Your lease at 123 Main Street expires on August 31, 2026"); and (f) the system displays the source document for visual confirmation.

4. Cross-Room Intelligence

A unique capability of the system is cross-room search and correlation: (a) a single query can retrieve related documents from multiple rooms (e.g., "What insurance covers my dog?" searches both Pets and Personal rooms); (b) the system identifies entity relationships across rooms (e.g., a family member mentioned in Family room documents is linked to their health insurance in Personal room); (c) aggregation queries compile information from multiple documents and rooms (e.g., "What are all my monthly insurance payments?" aggregates auto, health, renters, and pet insurance across rooms); and (d) the system proactively suggests related documents when one is accessed.

5. Zero-Knowledge Encryption Architecture

All documents are encrypted locally on the user's device using a key derived from the user's authentication credentials. The encryption scheme ensures: (a) documents are encrypted at rest using AES-256; (b) the encryption key never leaves the device; (c) no server-side component can access document plaintext; (d) the searchable index operates on encrypted metadata using techniques that preserve query capability without exposing document contents; and (e) backup and sync operations transfer only encrypted data.

Differentiation

Prior Art Analysis

Siri / Alexa / Google Assistant

Voice assistants that can initiate calls and answer general knowledge queries from cloud databases.

Lacks: No persistent document vault. No call participation after connection. No room-based organization. No OCR capture with voice wizard. No context-aware retrieval during calls. No zero-knowledge encryption.

Notion / Evernote

Text-first document management and note-taking platforms with folder-based organization.

Lacks: No voice-first interface. No AI call participation. No OCR-to-voice pipeline. No room-based hierarchical metaphor with cross-room intelligence. No zero-knowledge local encryption. No voice-guided document categorization.

Google Keep / Apple Notes

Lightweight note-taking with some OCR capability and basic voice input.

Lacks: No encrypted vault. No call integration. No contextual document retrieval during calls. No room-based organization. No voice-guided categorization wizard. No cross-room intelligence. No natural language query with voice response synthesis.
🚀
Novel Combination: No existing system combines (1) voice-first document capture with guided categorization, (2) hierarchical room-based encrypted storage, (3) natural language voice query across rooms, and (4) persistent AI participation in telephone calls with real-time document retrieval. Each component exists in some form, but the integration and the specific architectural approach (Mind Palace + persistent call participant + zero-knowledge vault) represents a novel system not found in prior art.
